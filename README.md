# Events API

REST API для управления мероприятиями.

# Технологии

- .NET 10
- C#
- OpenAPI 
- Swagger UI 

## Запуск

Собрать и запустить проект из папки 
Swagger доступен по адресу: `http://localhost:5161/swagger/index.html`

Запуск тестов:
- dotnet test

## Эндпоинты

| Метод  | Путь              | Описание                       | Статусы       |
|--------|-------------------|--------------------------------|---------------|
| GET    | /events           | Получить список всех событий   | 200           |
| GET    | /events/{id}      | Получить событие по ID         | 200, 404      |
| POST   | /events           | Создать событие                | 201, 400      |
| PUT    | /events/{id}      | Обновить событие               | 204, 400, 404 |
| DELETE | /events/{id}      | Удалить событие                | 200, 404      |
| POST   | /events/{id}/book | Создать заявку на бронирование | 202, 404      |
| GET    | /bookings/{id}    | Получить статус бронирования   | 200, 404      |

#### Параметры Get /events запроса (Query Parameters)

| Параметр   | Тип        | Обязательный | По умолчанию | Описание                           |
|:-----------|:-----------|:-------------|:-------------|:-----------------------------------|
| `title`    | `string`   | Нет          | —            | Фильтр по наименованию события.    |
| `from`     | `DateTime` | Нет          | —            | Фильтр по дате начала события .    |
| `to`       | `DateTime` | Нет          | —            | Фильтр по дате завершения события. |
| `page`     | `integer`  | Нет          | `1`          | Номер текущей страницы.            |
| `pageSize` | `integer`  | Нет          | `10`         | Количество элементов на странице.  |


## ❌ Обработка ошибок (Error Responses)
В случае неуспешного запроса API возвращает стандартный формат ошибок **Problem Details (RFC 7807)**.

### Коды ответов при ошибках

| Код                  | Причина                      | Пример сценария                             |
|:---------------------|:-----------------------------|:--------------------------------------------|
| `400 Bad Request`    | Ошибка валидации параметров. | Не передан обязательный параметр.           |
| `404 Not Found`      | Ресурс не найден.            | Запрос несуществующего события.             |
| `500 Internal Error` | Внутренняя ошибка сервера.   | Непредвиденный сбой в базе данных или коде. |


## Модели данных

### Событие

| Поле        | Тип      | Описание        |
|-------------|----------|-----------------|
| Id          | Guid     | Идентификатор   |
| Title       | string   | Наименование    |
| Description | int      | Описание        |
| StartAt     | DateTime | Дата начала     |
| EndAt       | DateTime | Дата окончания  |

### Бронь

| Поле        | Тип           | Описание                                                                                                  |
|-------------|---------------|-----------------------------------------------------------------------------------------------------------|
| Id          | Guid          | Идентификатор брони                                                                                       |
| EventId     | Guid          | Идентификатор события                                                                                     |
| Status      | BookingStatus | Статус бронирования: Pending (создана, ожидает обработки), Confirmed (подтверждена), Rejected (отклонена) |
| CreatedAt   | DateTime      | Дата и время создания брони                                                                               |
| ProcessedAt | DateTime      | Дата и время обработки брони                                                                              |

Данные хранятся в памяти и не сохраняются после остановки сервиса

Обработка заявок на бронирование выполняется в фоновом сервисе, при бронирование заявки находится в статусе Pending, 
после обработки в фоновом процессе заявка подтверждается или откланяется.
